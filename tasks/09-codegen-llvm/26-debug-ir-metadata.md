# 09-26: Debug IR metadata

**Phase:** 09-codegen-llvm    **Depends on:** 09-05, 09-12, 09-25    **Milestone:** M6    **Size:** Large

## Goal

Emit LLVM debug metadata in textual IR through `DIBuilder`, without depending
on the downstream driver object/link pipeline.

This is the backend slice of DWARF support. It proves that codegen can attach a
coherent source-level debug model to the LLVM module; object-level `.debug_info`
validation is split into `10-driver/13-debug-info-object-smoke.md`.

## Scope

- In:
  - `rcc_session::Options::debug_info` or equivalent internal switch.
  - `DICompileUnit`, `DIFile`, `DISubprogram`, and per-function lexical scope.
  - Scalar, pointer, array, and record type metadata sufficient for C99 user
    variables.
  - `DILocalVariable` / `DIParameterVariable` for named locals and parameters.
  - `dbg.declare` for stack slots generated by the current alloca-based backend.
  - Best-effort source locations from existing `Span` on function defs, locals,
    statements, and terminators.
- Out:
  - `-g` CLI parsing.
  - Object emission, linking, or `llvm-dwarfdump`.
  - DWARF v5 extensions, split DWARF, macro debug info, and optimized debug
    value tracking after mem2reg.

## Deliverables

- Debug info wrapper module in `rcc_codegen_llvm`, isolated from ordinary
  codegen logic.
- LLVM feature tests that opt in by setting `Session.opts.debug_info = true`.
- FileCheck-lite coverage for:
  - `!llvm.dbg.cu`,
  - `!DICompileUnit`,
  - `!DIFile`,
  - `!DISubprogram`,
  - `!DILocalVariable` / `!DILocalVariable(name: "param")`,
  - `call void @llvm.dbg.declare`.
- Negative coverage proving ordinary LLVM IR output remains free of debug
  metadata when `debug_info` is false.

## Acceptance

- With `debug_info = true`, `cargo test -p rcc_codegen_llvm --features llvm*`
  sees `!llvm.dbg.cu`, one `DISubprogram` per emitted function body, named
  locals/parameters, and `dbg.declare` calls in IR.
- With `debug_info = false`, the same fixture has no `!llvm.dbg.cu` and no
  `dbg.declare`.
- The LLVM module verifier still passes with debug metadata enabled.
- No project-wide default feature starts requiring LLVM.

## Follow-up Split

Object-level DWARF validation lives in
`tasks/10-driver/13-debug-info-object-smoke.md` and is intentionally downstream
of driver object emission and linker plumbing.
