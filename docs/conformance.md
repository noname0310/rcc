# Conformance dashboard

Tracks the pass/fail rate of `rcc` against every suite vendored in
[`third_party/MANIFEST.toml`](../third_party/MANIFEST.toml).

Regenerated by:

```bash
cargo xtask fetch-testsuites                # once per update
cargo run --release --package rcc_conformance --bin rcc_conformance_run -- \
    --rcc target/release/rcc \
    --suite c-testsuite
# writes docs/conformance.json + refreshes the table below
```

### Modes

`rcc_conformance_run` accepts a `--mode` flag (default `compile`):

```bash
# chibicc preprocessor-only gate (task 04-18, milestone M5)
cargo run --release --package rcc_conformance -- \
    --suite chibicc --mode preprocess
# full compile + link + run pipeline (milestone M6)
cargo run --release --package rcc_conformance -- \
    --suite chibicc --mode compile
```

`preprocess` runs `rcc --emit=pp -I<fixture-dir> <file>` per case and
checks the exit code; only preprocessor-focused fixtures
(`macro.c`, `typedef.c`, and `include.c` when vendored) are
discovered in that mode.

## Interpreting the columns

- **Discovered** — number of test cases the adapter enumerated.
- **Pass** — ran, stdout + exit code matched the expected values.
- **XFail** — listed in the suite's `xfail.toml`; treated as a pass by
  CI, but tracked separately so we can watch it shrink.
- **Fail** — ran, did not match. Blocks merge if in the required subset
  for the current milestone.
- **Skip** — intentionally not run (feature unsupported, adapter not
  yet implemented, GPL suite excluded without `--include-gpl`).

Some skips are semantic policy, not missing implementation. Cases whose stdout
depends on C99-unspecified evaluation order are listed in
`rcc_conformance::metadata` and demoted to **Skip** so differential runs do not
mistake a valid CFG evaluation-order choice for a compiler bug.

## XFail lists

Per suite:

- `third_party/testsuites/c-testsuite/xfail.toml`
- `third_party/testsuites/chibicc/xfail.toml`
- `third_party/testsuites/gcc-torture/xfail.toml`
- `third_party/testsuites/tcc-tests2/xfail.toml`
- `third_party/testsuites/llvm-test-suite/xfail.toml`

Format (see `rcc_conformance::xfail::XFailEntry`):

```toml
[[xfail]]
id     = "c-testsuite::00055"
reason = "bit-fields not yet implemented (M4)"
```
